How it works

A message in a topic is that session's next turn.

No orchestrator to manage. One supergroup with Topics enabled becomes your control surface — the General topic is the control plane, and every other topic is exactly one headless Claude Code session bound to a working directory.

1

Spawn from General

In the General topic, run /new <path> [name]. The bot creates a new topic and a live session with cwd set to that repo.

2

Talk in the topic

Whatever you type in a session topic is sent verbatim as the next turn. Output — text, tool calls, media — streams straight back into the same topic.

3

Run many in parallel

Each topic is independent and isolated. Two topics can even point at the same repo for parallel, separate lines of work.

Sessions run headless with bypassPermissions and are driven by the official claude-agent-sdk running the standalone claude CLI — one turn at a time, serialized per topic.

Features

Everything a phone-first coding cockpit needs.

Accurate to the box you run — nothing more, nothing hidden.

Parallel sessions

1 topic ⇄ 1 session, many at once, each isolated with its own working directory. One repo can back several topics.

Media, both directions

Send photos, files & video into a topic (images become vision input, everything lands in .tg-inbox/); the agent pushes media back via built-in tools.

Resumable across restarts

Every session's ID is persisted, so a bot restart lazily reconnects each session with full context — nothing is lost on reboot.

Container-isolated

The container boundary is what keeps bypassPermissions safe — sessions only reach the repos you mount at /workspace, never the rest of your host.

Batteries-included image

Common stacks (node, python, git, ffmpeg, ripgrep…) ship in the image; sessions apt/npm/pip install anything exotic at runtime.

Always-on & self-hosted

Runs anywhere Docker does with restart: unless-stopped. You own the bot, the box, and the data — no hosted lock-in.

Cheat sheet

Two places, a handful of commands.

Drive the whole thing from the General topic and from inside each session.

# General control plane
/new <path>Create a topic + session in that repo (relative to PROJECTS_ROOT or absolute).
/listList sessions and their status.
# session topic per session
/stopInterrupt the current turn.
/killEnd the session and close the topic.

Inside a topic, any message is the session's next turn — and a photo, file or video is downloaded to .tg-inbox/ and handed straight to the session.
